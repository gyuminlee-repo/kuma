## 주의사항 & 교훈
- 같은 파일을 여러 에이전트가 병렬 편집 시 race condition. Edit 호출 횟수 최소화 프롬프트 필수
- `EnterWorktree` 기본은 origin/main 기준 fresh checkout. 현재 HEAD 보존하려면 `git worktree add HEAD` 수동 생성
- 백그라운드 에이전트 stall (600s no-progress)는 빈번. 큰 파일(MainShell 533줄)은 stall 위험 높음
- main checkout에서 background에 spawn된 별도 에이전트가 동시에 main을 수정 중일 수 있음 — worktree 작업 완료 후 main의 미커밋 상태를 항상 git status로 확인
