# TAURI HOST SCOPE

Rust desktop host, bundled resources, sidecar integrity checks, and host-side tests.
Keep scientific logic in `kuma_core/` and RPC adaptation in `python-core/`.

## STRUCTURE

| Area | Role |
| --- | --- |
| `src/main.rs` | Thin binary entry |
| `src/lib.rs` | Tauri commands, app state, progress cache, ESM2 download |
| `src/sidecar.rs` | Python sidecar lifecycle |
| `src/sidecar_verify.rs` | Release SHA-256 verification |
| `tauri.conf.json` | Bundle resources, external binaries, updater, CSP |
| `samples/` | Explicitly bundled onboarding data |
| `tests/` | Rust host and protocol tests |

## CONVENTIONS

- Keep `main.rs` and the Rust host thin; place domain behavior in Python.
- Add every new bundled sample as an explicit file-to-file resource mapping in `tauri.conf.json`.
- Keep bundle files under `src-tauri/`; do not use `**` resource globs.
- Do not pass `--target` to `npx tauri build`; resource resolution depends on the default layout.
- `sidecar-hashes.json` is generated by `scripts/sidecar-hash*.mjs`; do not hand-edit it.
- Keep `package.json`, `tauri.conf.json`, `Cargo.toml`, and `pyproject.toml` versions aligned on release.

## GENERATED OR BUILD ARTIFACTS

- `target/`, `target-linux/`, and `target-win/`
- `.loophaus/`
- `binaries/`
- `resources/NOTICE.md`
- `sidecar-hashes.json`

## VERIFY

`cargo check` requires frontend assets and sidecar stubs when the expected binaries are absent.
Mirror the CI sequence when validating a clean checkout.

```bash
pnpm run build
cd src-tauri && cargo test
cd src-tauri && cargo check
pnpm sync:check
```
